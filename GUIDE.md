## Як почати?

Щоб розпочати процес розробки у вас має бути встановленний пакетний менеджер Nix

https://nixos.org/download/

1 крок:
```
curl --proto '=https' --tlsv1.2 -L https://nixos.org/nix/install | sh -s -- --daemon
```
2 крок:
```
echo "experimental-features = nix-command flakes" >> /etc/nix/nix.conf
```
3 крок:
В корні проекта (ros2_new/) прописуєтете 
```
nix develop
```
Та чекаєте коли всі залежності встановляться.
4 крок:
Приступаєте до розробки.


## Правила хорошого тону

Якщо Ви працюєте над інтеграцією чогось нового або суттєвим фіксом - створіть окрему гілку знаходячись на гілці з якої ви хочете підтянути все. 
Наприклад якщо Ви знайшли проблему в пакеті aruco_detector для вирішення якої пакет може потерпіти великі зміни - створіть гілку fix/aruco_{ваша будь яка назва, але бажано щоб вона була зрозумілою для інших (охарактеризовує суть проблеми)}.
Якщо це пакет - створіть гілку feature/{нащва пакету}

## GitOps

Не хвилюйтесь якщо у Вас є пробіли з гітом (або взагалі не вмієте ним користуватись). Цей проект є відмінним місцем для відтачування своїх навичків роботи в команді за допомогою Git. 

Особисто від себе можу порадити https://learngitbranching.js.org/. Тут ви дізнаєтесь про різницю між git merge та git rebase, як переключатись по коммітам, що таке revert, cherrypick та ще багато цікавого.
